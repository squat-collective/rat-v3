# Roadmap

Project-management view of RAT v3. **What are we doing, what have we done, what's queued, where are we.**

## Read in this order

1. **[current.md](current.md)** — what's in flight right now + the immediate next concrete step. Always read this first.
2. **[phases.md](phases.md)** — the phased plan, Phase 0 → Phase 5, with status per phase.
3. **[done.md](done.md)** — what's been completed, reverse chronological.
4. **[backlog.md](backlog.md)** — queued work not yet started.

## Maintenance

Rules for keeping the roadmap fresh: [CLAUDE.md](CLAUDE.md). Short version: **update after every working session that produces concrete output.** A stale roadmap is worse than no roadmap.
