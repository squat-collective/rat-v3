# Q02 maintainer defense log

The RAT v3 core maintainer/architect's responses to the four simulated Q02 reviewers
(architect, security, sre, ecosystem). For each question: **DEFENDED** (cite ADR +
file:line + rationale) or **CONCEDED** (honest gap). Discipline: concede, never bluff —
where the spike/core falls short of the frozen design, say so and point at the file.

Context for readers: Phase 0 (18-axis contracts) + Phase 1 (the core) are sealed locally
(`rat/1.5`, `rat/2.0`); the freeze is **local/unpushed**. The spike core in `core/`
implements the six things + cross-cutting enforcement, but several guarantees that froze as
prose MUSTs are stubbed in the spike (C2 channel auth, real backends, the marketplace/
issuance pipeline). Most findings below land in that "design frozen, enforcement not yet
wired" gap — which is exactly what Q02 was meant to surface before the freeze is published.

## Proactive corrections issued (to pre-empt strawman findings)

Sent before questions, to keep findings on real ground:
- **security/architect** — the capability-invoke gateway is **not a 7th core thing**: axis-generic, passthrough codec, parses zero payload bytes (`gateway.go:152-156`); routing by the `(rat.capability)` annotation (`gateway.go:134-150`). *(Not subsequently challenged.)*
- **all** — **tier-0** (state-backend, deployment-runtime, event-bus) is named honestly as bootstrap-critical / boot-selected (overview.md tier-0 callout; ADR-016 §2), not marketed as hot-swappable.
- **ecosystem** — the **two-reference rule (ADR-003) is a core-team burden, not a per-plugin-author one** — prevented a likely strawman.
- **architect** — the freeze is **evidence-backed** (the spike tried to break a frozen contract and didn't — ADR-013→015, reviews/10), while **Q01 (v3-vs-v2) and Q02 (first external eyes) are genuinely open** — conceded up front.

## Net-new concessions (real gaps NOT already in the backlog/ADRs)

These surfaced in Q02 and are **not** covered by an existing backlog item or ADR (or are only partially covered, with the specific gap untracked):

1. **A1 — additive semantic-field skew** has no negotiation mechanism; the crash-safety fields (ADR-012) shipped as plain fields, not as distinct capability URIs. A version-skewed idempotency consumer silently double-applies. *(Untracked.)*
2. **A2 — tenancy decision/enforcement mismatch**: `DECISION_KIND_SHARING` is decidable but **not actionable** (no delegated-tenant primitive in state/storage); hierarchical tenancy is inexpressible with the flat key. *(Untracked; R1 freeze-residual is a different concern.)*
3. **A3 — the keystone (gateway stamping, C1/C2/C3/C7) is not covered by the ADR-003 two-reference conformance gate** — one impl, unit-tested, no portable vector. The most irreversible surface has the weakest conformance. *(Untracked as a conformance obligation.)*
4. **S1 — no frozen contract obligation that an ArrowStream producer authenticate the presenting channel** before trusting the presented {caller,tenant}. The bytes-leg authn requirement is prose/honor-system, not a conformance MUST. *(E8 adjacent but does not state it.)*
5. **S2 — Arrow ticket has no `key_id`/rotation, and single-use is process-local** → replay across producer restart and across replicas within TTL. *(Untracked.)*
6. **S3 — conformance trust root has no revocation and no attestation expiry**, and no multi-authority scoping → a conformed-then-vulnerable plugin stays trusted forever; one leaked key = full trust. *(C8 covers signing in general; revocation/expiry/scoping specifics untracked.)*
7. **R1 — lease has no transient-error-vs-lost distinction** (`bool`-only CAS) → a backend blip churns leadership / stalls reconvergence on every replica. *(sre#4 covered only backoff/jitter+thrash.)*
8. **R2 — `Degraded` is terminal with no resume path and no transition event/metric** → a transient outage kills a plugin until a human restarts the core, silently. *(sre#8 carries the metric only; the terminal-vs-capped decision + resume + event are untracked.)*
9. **R4/F7 — both "enforcing" deployment-runtimes silently drop the frozen `LaunchSpec` resource limits** (no `--memory`/`--cpus`/`--pids-limit`) → uncontained runaway/fork-bomb → cross-tenant resource starvation. *(Untracked; pure impl gap on an already-frozen field.)*

Plus one concrete **doc-drift sub-finding** (E1): `plugins/README.md` promised by ADR-006 D2 does not exist, and the 2 `contracts/examples/` manifests don't match the runnable `plugins/` dirs.

**Already-tracked (conceded but NOT net-new):** R3 (DR/upgrade — prospective ADR-015/016, undecided); E2 (inner dev loop — Phase 2 `rat dev --plugin`); E3 marketplace half (C8 + the CONTRACT.md honesty banner already admit self-assertion).

## Maintainer's read

The frozen *contracts* held up well (the spike's whole purpose — ADR-013/015 — and Q02 found no demand to reopen the data-plane wire). The damage is concentrated where the project already knew it was thin: **enforcement that froze as prose with no conformance vector** (A3, S1), **the bytes-leg honor-system** (S1, S2), **the spike's stubbed cross-cutting seams** (R1, R2, R4, C2), and **trust/PKI lifecycle + ecosystem on-ramp** (S3, E1). The two highest-value findings are **A3** (the conformance gate that justifies the freeze doesn't cover the riskiest surface) and **S1** (a missing *frozen-contract obligation*, expensive to add post-publish). Both are best fixed while the freeze is still local — which is precisely the window Q02 was scheduled to protect.
